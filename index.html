git - system kontroli wersji

  • statusy plików i polecenie git status
  • commitowanie zmian do repozytorium git commit -m "opis commita"
  • cofanie ze stage-a git reset HEAD readme
  • cofanie zmian z kodu git checkout -- readme
  • usuwanie plików z repozytorium - plik zostaje na dysku git rm --cached readme
  • historia zmian git log --oneline
  • historia 3 ostatnich zmian git log -3 --oneline
  • zmiana nazwy pliku z readme na readme.txt git mv readme readme.txt
  • ****************************************
  • jakie mamy branche? git branch
  • utworzenie brancha na podstwie rewizji git checkout f198465 -b stan_bez_branchow
  • utworzenie brancha git branch nazwa lub git checkout -b nazwa
  • przelaczenie na brancha git checkout master
  • historia dla konkretnego brancha git log test --oneline
  • jestem w master i robie merge z brancha test (zmiany sa tylko w test wiec nie ma konfilktow) git merge test
  • cofniecie ostatniej operacji merge git reset HEAD~1 --hard
  • ladne ogladanie branchow git log --oneline --graph --all --decorate
  • usuwanie branchy (dla branchow scalonych) git branch -d test2
  • usuwanie branchy (dla branchow nie scalonych) git branch -D test2
  • ****************************************
  • poprawa opisu commita git commit --amend
  • dodanie taga git tag 1.0
  • usuniecie taga git tag -d 1.1
  • ****************************************
  • podlaczenie zdalnego repozytorium pod lokalnyalias origin git remote add origin https://github.com/pmswiatek/NaukaGit.git
  • lista podlaczonych zdalnych repozytoriow git remote lub git remote -v
  • wyslanie danych do zdalnego repozytorium (master -> origin) git push -u origin master
  • pobranie danych ze zdalnego repozytorium git fetch origin master
  • sprawdzenie zmian miedzy lokalnym i zdlanym repozytorium git remote show origin
  • wyslanie ponownie zmian do zdalnego repo git push origin master
